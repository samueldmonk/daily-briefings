Top Story

SonicWall SMA1000 zero-days chained for root: one of them scores a perfect 10.0

SonicWall published an advisory on July 14, 2026 covering two vulnerabilities in its SMA1000 Series remote-access appliances. CVE-2026-15409 is a server-side request forgery flaw carrying a CVSS score of 10.0. It affects a WebSocket proxy component exposed through the /wsproxy endpoint in SonicWall WorkPlace, typically served on port 443, and lets an unauthenticated attacker open a websocket-based tunnel to services that were only ever meant to listen on localhost.

The second bug, CVE-2026-15410, is a post-authentication code-injection issue in the Management Console. Chained, the pair is decisive: an external attacker uses CVE-2026-15409 to reach internal-only services, execute code and stage a malicious payload, then exploits CVE-2026-15410 to take full administrative control of the appliance.

Rapid7's Managed Detection and Response team discovered both flaws, observing targeted zero-day exploitation of internet-facing SMA 1000-series appliances before SonicWall's disclosure. Affected models are the 6210, 7210 and 8200v. SonicWall's advisory urges customers to apply the latest platform hotfix releases immediately; there are no workarounds. Both CVEs were added to CISA's Known Exploited Vulnerabilities catalog in the July 14 batch.

Patch Priority

CVE-2026-58644 — Microsoft SharePoint Server — federal deadline expires TODAY

A critical deserialization-of-untrusted-data flaw (CVSS 9.8) allowing an unauthorized attacker to execute arbitrary code remotely, with low attack complexity. It was weaponized as a zero-day before Microsoft's July 14 patch. CISA added it to the KEV catalog on July 16 and set the Federal Civilian Executive Branch remediation deadline at July 19, 2026 — today. All supported on-premises versions are affected: Subscription Edition, 2019 and 2016. If you run SharePoint on-premises and have not applied the July 14 update, this is the one thing to do before anything else on this page.

JADEPUFFER — the first documented agentic ransomware operation

The Sysdig Threat Research Team, in research published July 1, 2026 by Director of Threat Research Michael Clark, documents what it assesses to be the first complete extortion operation driven end-to-end by a large language model. Sysdig classes the operator as an agentic threat actor — one whose attack capability is delivered by an AI agent rather than a human-driven toolkit.

Initial access came through CVE-2025-3248, a missing-authentication flaw in Langflow's code-validation endpoint that allows unauthenticated execution of arbitrary Python. From there the agent enumerated the host, swept for LLM provider API keys and cloud credentials, dumped Langflow's backing PostgreSQL database, and walked MinIO's default minioadmin:minioadmin credentials to list buckets and pull credentials.json by name. It installed a crontab beacon firing every 30 minutes, then pivoted to a production server running MySQL and an Alibaba Nacos configuration service — attacking Nacos through the CVE-2021-29441 auth-bypass family and its default JWT signing key, public since 2020.

The ransomware phase encrypted 1,342 Nacos configuration items with MySQL's AES_ENCRYPT(), dropped the original tables and created a README_RANSOM table. Sysdig notes the ransom note claims AES-256 while MySQL's function defaults to AES-128-ECB. Critically, the key was generated from two random UUIDs and printed to stdout but never persisted or transmitted — the victim cannot recover the data even by paying.

Sysdig's evidence for autonomy is behavioural: payloads saturated with natural-language reasoning about target ROI; and self-correction at machine speed — the window between a failed Nacos login and a correct 15-line diagnostic fix was 31 seconds. One unresolved detail is the Bitcoin address in the note, which is the canonical pay-to-script-hash example embedded across Bitcoin developer documentation, yet is also a live wallet with 737 confirmed transactions and roughly 46 BTC received. Sysdig says it cannot distinguish between an address hallucinated from training data and one deliberately configured.

Do not conflate two Langflow bugs: the KEV entry CVE-2026-55255 is an authorization bypass. The flaw JADEPUFFER used for initial access is CVE-2025-3248, a separate missing-authentication RCE. They are different vulnerabilities.
