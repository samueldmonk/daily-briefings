Top Story

SleeperGem: hijacked dormant RubyGems accounts are shipping a backdoor that deliberately skips your CI and goes for the developer's laptop

Researchers have flagged a software supply-chain attack codenamed SleeperGem after three malicious gems were published to RubyGems to serve follow-on payloads. The rogue releases are git_credential_manager (versions 2.8.0, 2.8.1, 2.8.2 and 2.8.3, published July 18, 2026), Dendreo (1.1.3, 1.1.4) and fastlane-plugin-run_tests_firebase_testlab (0.3.2).

"Each malicious release is a loader," StepSecurity said in its analysis. "It fetches a second stage from an attacker controlled Forgejo host, checks whether it is running in a build system and skips if it is, and on a developer machine it drops a native daemon and installs persistence."

The name comes from the tradecraft. git_credential_manager impersonates Microsoft's official Git Credential Manager; the other two had been dormant for years before their sudden updates — Dendreo was last touched on October 24, 2020 and the fastlane plugin had been inactive since March 9, 2019. All three releases went straight to the registry with no matching commit or tag in the source projects. "A RubyGems account that has gone quiet for six or seven years doesn't look risky to anyone," Aikido Security researcher Charlie Eriksen said. "That's exactly the profile worth taking over."

Blast radius is wider than three packages. git_credential_manager was added as a dependency to five packages — Dendreo, fastlane-plugin-run_tests_firebase_testlab, slackHtmlToMarkdown, seo_optimizer and array_fast_methods — which spreads the payload to existing users. All but the fastlane plugin sit under one maintainer account (LR-DEV); the plugin belongs to another (pinkroom), indicating more than one account was compromised.

The evasion is the notable part. On install the malware scans for roughly 30 environment variables associated with GitHub Actions, GitLab, CircleCI, Travis, Jenkins and Vercel — and exits if it finds any. That is an intentional attempt to avoid ephemeral CI runners and land on a real developer machine, where credentials live longer. Merely requiring the library fires the code, which pulls a shell script and a native binary from a public Forgejo instance (git.disroot[.]org/git-ecosystem); on Windows the payload runs via PowerShell. Version 2.8.2 stages the payloads, while 2.8.3 launches the binary as a background daemon and establishes persistence through a cron entry and a systemd user service, then queries the sudo and wheel groups. "If the user can run sudo without a password, the script re-runs itself as root, and when it runs as root it plants a setuid root copy of the system shell at a path chosen to mimic a networking utility," StepSecurity said.

If you installed any of these gems, treat the machine and its secrets as compromised. Remove the dropped daemon at ~/.local/share/gcm/, erase the cron and systemd persistence, check for a setuid shell at /usr/local/sbin/ping6, and rotate every credential the host touched.

Patch Priority

Do this first: SharePoint CVE-2026-58644 — the federal deadline has already passed

CVE-2026-58644 is a CVSS 9.8 remote-code-execution flaw in Microsoft SharePoint that is in CISA's Known Exploited Vulnerabilities catalog with a remediation date of July 19, 2026 — meaning federal civilian agencies are already past due and everyone else is patching against confirmed in-the-wild exploitation. Internet-reachable SharePoint should be treated as the highest-priority item on the estate today.

Running a close second: CVE-2026-15409 in SonicWall SMA1000, scored at the maximum 10.0 with a public proof-of-concept available. It is in the KEV catalog from the July 14 batch, but no CISA-stated due date was confirmed in this run, so none is asserted here — the exploit maturity alone justifies emergency handling.

This box, the KEV section below and every countdown on this page use the same verified dates. Where CISA has published an addition but we could not confirm the individual due date, the item says so rather than guessing.

Threat Actor Spotlight

"888" — the broker selling Accenture's source code on PwnForums

An actor operating under the alias 888 listed 35 GB of claimed Accenture data on the PwnForums marketplace, describing the haul as source code, RSA and SSH keys, Azure access keys and personal access tokens, and configuration files. The listing pattern — credentials and cloud tokens bundled with code, offered to the highest bidder rather than encrypted for ransom — is characteristic of access brokers who monetise a foothold instead of running it themselves.

Accenture's response, verbatim: "We are aware of this isolated matter, and we have remediated its source. There is no impact to Accenture operations and service delivery." The company has not confirmed the volume or the type of data, so the 35 GB figure and the contents list remain the seller's claims, not verified fact.

Breaches & Incidents

BreachAI platform

Hugging Face confirms internal datasets and credentials were hit

Disclosed July 16; on July 20 Hugging Face confirmed the intrusion reached internal datasets and credentials and urged users to act. The disclosure describes "many thousands of individual actions across a swarm of short-lived sandboxes." Detection came from an anomaly-detection pipeline using LLM-based triage, with analysis agents working over more than 17,000 recorded events — hours of work instead of days. Notably, frontier models behind commercial APIs refused the forensic requests because the payloads were exploit code and C2 artifacts; the reconstruction ran on open-weight GLM 5.2 on Hugging Face's own infrastructure.

NewIncident

Progress orders ShareFile customers to shut down Storage Zone Controllers

On July 10, Progress Software told ShareFile customers to power off the Windows servers running their Storage Zone Controllers, confirming to The Hacker News that it is responding to a "credible external security threat." Progress temporarily disabled access to affected accounts "out of an abundance of caution" and says it has no indication of unauthorized access to any ShareFile account or data. It has not said what the threat is, who is behind it, or when systems can come back. Only self-hosted Storage Zone Controllers are affected — cloud-only ShareFile accounts are not. Ordering a full shutdown rather than a patch is the unusual part.

Supply chain

Jscrambler npm packages backdoored via a stolen publishing credential

Version 8.14.0 shipped on July 11 with a preinstall hook dropping a native binary for Windows, macOS and Linux; Socket flagged it six minutes after publication. Jscrambler confirmed a compromised npm publishing credential. Five malicious versions — 8.14.0, 8.16.0, 8.17.0, 8.18.0 and 8.20.0 — went out over roughly three hours, and in 8.18.0 and 8.20.0 the dropper moved into the main code so npm install --ignore-scripts no longer stops it. The Rust infostealer targets AWS/Azure/GCP credentials, MetaMask, Phantom, Exodus, Bitwarden, browser passwords and cookies, Discord/Slack/Telegram/Steam, and AI-tool configs. The clean release is 8.22.0.

Ransomware

Blackfield demands $2 million from Nidec

The Blackfield gang compromised Nidec Chaun Choung Technology, the Japanese motor maker's Taiwanese subsidiary, with the attack detected on Monday, June 22, 2026. The crew claims more than 2 TB of employee, financial, procurement, manufacturing, legal and IT records and has given the company more than 15 days to negotiate.

ExtortionDefense

Gentlemen gang posts Indra Group to its leak site

Indra Group, the Spanish defense and aerospace contractor and a member of NATO's cyber coalition, had one subsidiary affected. The Gentlemen gang listed it on June 30 with a nine-day deadline. Indra says the incident is contained and service continuity has been maintained.

Credential theft

FortiBleed: 74,000 FortiGate credentials tied to ransomware deployment

A set of 74,000 stolen FortiGate credentials has been linked to the INC and Lynx ransomware groups — described as the first confirmed mass FortiGate credential theft connected directly to ransomware deployment rather than resale alone.
