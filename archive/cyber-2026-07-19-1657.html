Top Story

wp2shell goes from theory to weapon: public exploits land for WordPress core pre-auth RCE

Two days after WordPress shipped emergency releases, the situation around wp2shell has changed materially. BleepingComputer reported on July 18 that multiple public proof-of-concept exploits have been published on GitHub, and security firm watchTowr says it is already seeing the consequences.

"The watchTowr team is already seeing PoC exploits in circulation, and we are beginning to see the first signs of in-the-wild exploitation," watchTowr CEO Benjamin Harris told BleepingComputer. He framed why this one matters: "a highly impactful, unauthenticated SQL injection or remote code execution vulnerability in WordPress core is actually fairly rare."

The attack is a chain of two flaws, not one. CVE-2026-63030 is a REST API batch-route confusion bug introduced in WordPress 6.9; CVE-2026-60137 is a SQL injection in the author__not_in parameter of WP_Query. Chained, they take an anonymous HTTP request to code execution against a stock install with no plugins. The full chain affects 6.9.0–6.9.4 and 7.0.0–7.0.1, fixed in 6.9.5 and 7.0.2 (and 7.1 Beta 2). The SQL injection alone also reaches 6.8.0–6.8.5, but cannot be chained to RCE there because the batch-route bug arrived in 6.9.

Severity ratings diverge and both are published here rather than one being picked: the GitHub Security Advisory classifies the issue as Critical, while the CVE record carries a CVSS base score of 7.5. Note also that the exploits are not all equivalent — BleepingComputer reports some public PoCs extract password hashes via SQL injection and then require cracking an administrator password, while others claim true pre-auth RCE with no credentials, which matches Searchlight Cyber's description. Searchlight has withheld technical details and published a checker at wp2shell.com instead; Cloudflare has deployed WAF rules for both CVEs across all plans including free accounts, while cautioning that "WAF protections reduce exposure while customers update, but they are not a substitute for patching."

One number needs care: Searchlight's "more than 500 million websites" figure is the total WordPress install base, not the vulnerable population — the flawed code exists only from 6.9, which shipped in December 2025.

Patch Priority

Today is the federal deadline for CVE-2026-58644 — and WordPress cannot wait either

First: Microsoft SharePoint Server, CVE-2026-58644 (CVSS 9.8). A deserialization-of-untrusted-data flaw giving unauthenticated remote code execution, exploited as a zero-day before the July 14 Patch Tuesday fix. CISA added it to the KEV catalog on July 16 with an FCEB remediation deadline of today, July 19, 2026. It affects all supported on-premises versions — Subscription Edition, 2019 and 2016. The two Fortinet FortiSandbox command-injection flaws added the same day share the same deadline.

Then: WordPress. Update every internet-facing site to 6.9.5 or 7.0.2 immediately. Public exploits are circulating and early in-the-wild activity is being reported. WordPress has forced automatic updates for affected installations that have auto-update enabled — but it has not confirmed whether that push reaches sites with auto-updates disabled, so verify the version you are actually running rather than assuming. If you genuinely cannot patch, block both /wp-json/batch/v1 and ?rest_route=/batch/v1 at the WAF — Searchlight is explicit that blocking only the path leaves the query-string route open.

Threat Actor Spotlight

InfostealerMalware-as-a-serviceNew

ACR Stealer — Microsoft flags a surge against enterprise customers

Microsoft published research on July 16 warning of increased ACR Stealer activity observed by Defender Experts across customer environments from late April to mid-June 2026. The operation is a malware-as-a-service offering believed to be a rebrand of Amatera Stealer, and the delivery chain is notable for how ordinary it looks: ClickFix social-engineering lures that persuade the user to run something themselves, followed by WebDAV servers and the mshta.exe HTML Application host to stage the payload.

What it takes is the modern crown jewels — browser-stored credentials, session tokens and authentication artifacts, plus sensitive documents including Microsoft 365 files. Stolen tokens are the real danger: they enable account compromise and cloud access without ever touching a password. Microsoft's guidance leans on blocking low-reputation and newly registered domains and using application-control rules to stop mshta.exe, PowerShell, Python and rundll32.exe launching remote content from user-writeable paths.

Breaches & Incidents

NewExtortionEnergy

Ecopetrol: data tied to 3,300 accounts stolen, ransomware attempt blocked

Colombia's state-controlled energy company disclosed on July 17 that an external actor accessed cloud file-storage environments across 15 subsidiaries and stole data linked to roughly 3,300 user accounts. Ecopetrol says it prevented an attempted ransomware deployment. The attacker has not been identified, has communicated extortion demands and has threatened publication, though no data had been released as of the disclosure. The company reports no critical disruption to operations or production, but warns the incident could still have a material adverse effect while it assesses exposure.

NewRansomwareManufacturing

Coca-Cola halts U.S. Fairlife production after ransomware

Coca-Cola disclosed that a ransomware attack on its Fairlife dairy subsidiary reached production-related systems, and that Fairlife's U.S. production operations are temporarily suspended. Canadian operations are unaffected. This is the rare incident where the operational impact is the story rather than the data: Fairlife is a billion-dollar-scale brand, and the company has activated incident response and business continuity protocols and notified law enforcement. Coca-Cola has not said whether data was stolen or whether it is being extorted, and no ransomware group has claimed the attack.

NewThird partyProfessional services

Ernst & Young: client tax documents taken via a support-ticket platform

EY is notifying clients that an unauthorised third party accessed a support ticket platform used by its IT staff between March 28 and April 12, 2026, downloading documents before the intrusion was spotted on April 23. The exposed material includes personal information tied to individuals' investment holdings with EY's institutional clients and financial information used to prepare tax filings, and may extend to Social Security numbers, financial account codes and card details. EY filed breach notifications with the California Attorney General on July 15 and is offering 24 months of Experian identity monitoring, with enrolment open until October 31. It says it has no current evidence of misuse.

NewUnpatchedWindows

LegacyHive zero-day PoC lands hours after Patch Tuesday

A researcher using the handle Nightmare Eclipse published a proof-of-concept for a local privilege-escalation flaw in the Windows User Profile Service within hours of Microsoft's July 2026 updates. It abuses arbitrary registry hive loading, letting a standard user mount another user's hive — including an administrator's — into their own classes root. It works on fully patched July 2026 systems across supported Windows 10, Windows 11 and Windows Server builds, and has no CVE assigned. Weaponisation is not trivial: the released PoC requires a second standard-user credential plus a third username. Microsoft says it is investigating.
