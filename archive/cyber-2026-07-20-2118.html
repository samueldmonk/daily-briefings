Top Story

Microsoft dissects the @asyncapi npm compromise: the payload runs on import, not install

On July 14, 2026, Microsoft Threat Intelligence identified a coordinated supply-chain compromise of the @asyncapi npm organization. Five package versions across four package names were republished within roughly ninety minutes, each carrying the same maliciously injected loader: @asyncapi/specs (both the 6.11.2-alpha.1 prerelease and the 6.11.2 stable release), @asyncapi/generator@3.3.1, @asyncapi/generator-components@0.7.1 and @asyncapi/generator-helpers@1.1.1.

The operational detail that matters: unlike the common postinstall-hook pattern, this campaign executes at module-load (import/require) time. Microsoft states plainly that because the trigger is an import rather than an install script, the common npm install --ignore-scripts mitigation does not neutralize it. The affected packages declared no preinstall, install or postinstall hooks at all — a deliberate choice that bypasses hook-focused scanners.

The entry point was a GitHub Actions “pwn request”: a pull_request_target workflow in asyncapi/generator checked out an untrusted PR head commit while holding a privileged token, exposing the asyncapi-bot personal access token. The attacker then rode the project’s legitimate OIDC release workflows — so all five malicious versions shipped with valid provenance attestations that correctly identified the real repositories, commits and workflows, even though the triggering commits were unauthorized. Because @asyncapi/specs is a transitive dependency of numerous AsyncAPI tooling packages, the blast radius covers developer workstations, CI/CD pipelines, container builds and production services.

Stage two fetches an encrypted sync.js (~8.2 MB) over IPFS into an OS-specific “NodeJS” masquerade directory, then initialises the Miasma modular runtime (identified as M-RED-TEAM v6.4, campaign miasma-train-p1) with active C2, persistence and decentralised fallback discovery via Nostr, Ethereum, BitTorrent DHT, libp2p and IPFS. Credential-harvesting, propagation, AI-tool poisoning, metamorphic generation and sandbox-evasion modules were implemented but disabled in the analysed build. Microsoft’s guidance: pin @asyncapi/specs 6.11.1 or earlier, generator 3.3.0, generator-components 0.7.0 and generator-helpers 1.1.0; purge npm and Yarn caches; hunt for sync.js; block 85.137.53[.]71 on ports 8080, 8081 and 8091; and rotate every credential reachable from any environment that imported an affected version.

This is the second npm supply-chain compromise carrying the same “--ignore-scripts won’t save you” property inside a week — the Jscrambler credential compromise (below) moved its dropper into main code and the CLI in versions 8.18.0 and 8.20.0. Jscrambler is demoted to a card here on recency, not significance.

SonicWall SMA 1000: CVE-2026-15409 (CVSS 10.0) — federal due date was July 17

CVE-2026-15409 is a critical server-side request forgery flaw in the SMA1000 Appliance Work Place interface allowing a remote, unauthenticated attacker to force the appliance to make requests to unintended locations. Rapid7’s MDR team observed active, targeted zero-day exploitation of internet-facing SMA 1000-series appliances, with attackers chaining it to CVE-2026-15410 (post-authentication code injection in the Appliance Management Console, CVSS 7.2) for full device takeover. Affected firmware: SMA 1000 Series 12.4.3 and 12.5.0.

Fix: platform-hotfix 12.4.3-03453 or higher, or 12.5.0-02835 or higher, per SonicWall’s product notice. CISA added both CVEs to the KEV catalog on July 14 with a remediation date of July 17, 2026 under BOD 26-04 — agencies unable to mitigate were directed to discontinue use of the product. That date is now 3 days overdue.

The same July 17 date applies to CVE-2026-56164 (SharePoint). The KEV section below uses these identical verified dates — no separate deadline is asserted anywhere on this page.

RansomwareNewRust

Spirals — break-in to blackout in under 24 hours

A previously unknown ransomware strain used against an IT services company in South Asia, where the operators went from initial access to data theft and full network encryption in less than 24 hours. Entry was an internet-facing IIS web server and an uploaded ASP.NET web shell; commands were run through the IIS worker process to open an interactive session, then privileges were escalated with a UAC bypass, RDP was switched on, and a payload disabled Windows Defender and stopped services tied to 23 backup, database and virtualization products. Spirals is written in Rust, encrypting each file with a separate AES-128 key wrapped under an attacker-controlled ECDH P-256 public key; files over 5 MB are chunked for speed. The note, RECOVERY_SECTION.log, points victims to a Tor negotiation site and threatens to leak stolen data within six days. Seen on only one victim network so far — but researchers assess the tradecraft as that of skilled operators capable of far wider campaigns.

Breaches & Incidents

HealthcareNew62.2M

Conduent victim count tops 62.2 million

The SafePay ransomware breach at business-services provider Conduent now stands at 62.2 million individuals — the third-largest US healthcare breach on record behind Change Healthcare (192.7M, 2024) and Anthem (78.8M, 2015). Conduent discovered on January 13, 2025 that attackers had been inside its network since October 21, 2024. The count escalated in stages: ~10.5M when first reported, revised to 25.5M in February, and finally 62.2M by June 2026. Conduent accrued $25 million in direct breach-response costs in Q1 2025 alone, with a further $16 million anticipated through Q1 2026.

RansomwareManufacturing

Coca-Cola’s fairlife confirms ransomware, US production paused

Coca-Cola’s US dairy subsidiary fairlife confirmed a ransomware attack that temporarily halted production across the United States. Per the SEC Form 8-K disclosed July 16: unauthorized access including production-related systems; US production temporarily suspended with Canada unaffected; law enforcement notified. The company states product quality and safety were not impacted. The attackers have not been identified, data theft is not confirmed, and no group has claimed the intrusion.

Professional servicesNotification

Ernst & Young: client tax filings exposed

Anomalous activity was spotted April 23, with actual attacker access dated March 28–April 12. The California AG filing landed July 15, one of filings in four states. Exposure includes Social Security numbers, financial account codes, investment holdings and the contents of client tax filings. Affected individuals get 24 months of Experian monitoring and must enrol by October 31, 2026.

Supply chainnpm

Jscrambler npm packages backdoored via stolen publishing credential

Version 8.14.0, published July 11, ran a Rust infostealer from a preinstall hook across Windows, macOS and Linux builds. Socket flagged it six minutes after publication and ties five malicious versions to one actor over roughly three hours (8.14.0, 8.16.0, 8.17.0, 8.18.0, 8.20.0). Jscrambler confirmed the cause as a compromised npm publishing credential. Critically, in 8.18.0 and 8.20.0 the dropper moved into the package’s main code and CLI, firing on import or run — --ignore-scripts does not stop those. Targets span AWS/GCP/Azure credentials, MetaMask/Trust Wallet/Coinbase Wallet/Phantom/Exodus, Claude Desktop/Cursor/Windsurf/Factory/Zed/VS Code, Discord/Slack/Telegram Desktop, browsers and OS keyrings. 15,800+ weekly downloads. Clean release is 8.22.0; deprecation still leaves bad versions installable by exact version.
