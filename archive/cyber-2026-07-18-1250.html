Top Story

wp2shell: an anonymous HTTP request can run code on a default WordPress site

WordPress shipped 6.9.5 and 7.0.2 on Friday, July 17, closing a pre-authentication remote code execution flaw in core. There are no preconditions: a bare install with zero plugins is exploitable by an anonymous request. Adam Kues at Assetnote — Searchlight Cyber's attack surface management arm — found it and reported it through WordPress's HackerOne program, publishing under the name wp2shell.

Affected ranges are 6.9.0 through 6.9.4 (fixed in 6.9.5) and 7.0.0 through 7.0.1 (fixed in 7.0.2). WordPress's release post describes the finding as "a REST API batch-route confusion and SQL injection issue leading to Remote Code Execution." 7.1 beta2 carries the same fix; 6.8.6 addresses a separate SQL injection reported by a different team. WordPress enabled forced updates through its auto-update system but has not said whether that push reaches sites with auto-updates turned off — check what you are actually running.

The scanning problem: neither advisory carries a CVE ID or a CVSS score, and no CVE record had appeared by July 18. CVE-keyed scanners and inventories will not flag this one, and CISA needs a CVE before it can add anything to KEV. Track it by version number. Searchlight's "over 500 million websites run WordPress" figure is the total install base, not the vulnerable population — the flawed code exists only from 6.9 onward, which shipped December 2, 2025. No exploitation attempt has been reported as of July 18, and the firm is withholding technical details, offering a checker at wp2shell.com instead.

CVE-2026-58644 — Microsoft SharePoint Server (CVSS 9.8). Deserialization of untrusted data allowing an unauthorized attacker to execute arbitrary code; exploited in the wild as a zero-day before the July 14 fix. CISA added it to the KEV catalog on July 16 with an FCEB remediation deadline of July 19, 2026. Affected: Subscription Edition, 2019, and Enterprise Server 2016. Remotely exploitable, low attack complexity.

The same deadline covers CVE-2026-25089 and CVE-2026-39808, the two Fortinet FortiSandbox OS command-injection flaws CISA added the same day on evidence of active exploitation. If you run SharePoint on-prem or FortiSandbox, these three come before everything else on this page.

Then WordPress. wp2shell has no CVE and no public signature, so it will not appear in vulnerability-management dashboards — inventory by version and confirm 6.9.5 / 7.0.2 actually landed.

Threat Actor Spotlight

ShinyHunters

The extortion gang added Abbott Laboratories to its data leak site this week, and Abbott confirmed unauthorized access to legacy Exact Sciences systems in its Cancer Diagnostics business. ShinyHunters told BleepingComputer it got in via a vishing attack on several Abbott employees in mid-June, compromising a Microsoft Entra single sign-on account.

That is the group's signature play: since last year it has run social-engineering campaigns against employees' Microsoft Entra, Okta and Google SSO accounts, then pivoted to connected SaaS — Salesforce, Microsoft 365, Google Workspace, SAP, Slack, Adobe, Atlassian, Zendesk, Dropbox. It has increasingly targeted medtech, including Medtronic, OneMedical and AdaptHealth, and BleepingComputer reports it was behind the iRhythm breach. Defensive takeaway: the initial access is a phone call, not a payload — help-desk identity verification is the control that matters.

Breaches & Incidents

NewExtortionHealthcare

Abbott investigating two separate cyber incidents

Abbott confirmed unauthorized access to a limited number of legacy Exact Sciences systems in its Cancer Diagnostics business after ShinyHunters listed it, saying the incident does not affect operations, products or lab work and is not expected to be material. ShinyHunters initially threatened to publish after July 18, then extended to July 21, and claims over 30 million rows of customer PII including more than a million Social Security numbers — claims BleepingComputer has not independently verified. Separately, an actor called ShadowByt3$ claims it breached the LabCentral customer portal on July 4 using compromised customer credentials; Abbott says that portal holds only publicly available technical documentation. Neither actor has released data.

NewData theftProfessional services

Ernst & Young discloses breach of IT support platform

EY notified clients that an unauthorized third party accessed a support ticket platform used by its IT staff between March 28 and April 12, 2026, downloading documents belonging to a number of clients. The firm filed breach notifications with the California Attorney General's office on July 15. Compromised documents contained personal information tied to individuals' investment holdings with EY's institutional clients along with financial information used to prepare tax filings; exposed fields may include Social Security numbers, financial account codes and credit and debit account information.

TakedownProxy abuse

Google disrupts NetNut residential proxy network

Google moved against the NetNut residential proxy network, which spanned roughly 2 million home devices. Residential proxies launder attacker traffic through consumer IP space, which is what makes credential-stuffing and scraping campaigns hard to filter on reputation alone.

SpywareTargeted

EU Parliament member investigating spyware hacked with Pegasus

A member of the European Parliament who was investigating commercial spyware was themselves compromised with Pegasus — a reminder that mercenary spyware continues to be aimed at oversight and civil-society targets, not just at conventional intelligence objectives.

CryptoExploited

'Ill Bloom' flaw drained over $5 million from crypto wallets

Attackers exploited the Ill Bloom weakness in wallet recovery-phrase generation to drain more than $5 million from cryptocurrency wallets. The root cause is weak randomness in seed-phrase generation, which makes affected phrases guessable rather than requiring any compromise of the user's device.
