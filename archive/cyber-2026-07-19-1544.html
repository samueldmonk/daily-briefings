Top Story

wp2shell gets its CVE: WordPress core pre-auth RCE is now CVE-2026-63030

On July 17 a GitHub Security Advisory (GHSA-ff9f-jf42-662q) was published for CVE-2026-63030, the unauthenticated remote-code-execution flaw in WordPress core that researchers named wp2shell. Tenable records the CVE as published July 17 and updated July 18, sourced to Mitre and NVD — closing the gap flagged in earlier editions, when neither advisory carried a CVE ID and scanners had nothing to match on.

The bug is a REST API batch-endpoint route confusion which, combined with the author__not_in WP_Query SQL injection tracked as CVE-2026-60137, lets an attacker reach SQL injection and then remote code execution. It affects WordPress 6.9.0 through 6.9.4 and 7.0.0 through 7.0.1; the fixes shipped July 17 in 6.9.5 and 7.0.2, and the same fix is carried in 7.1 Beta 2.

Adam Kues at Assetnote — Searchlight Cyber's attack surface management arm — found the flaw and reported it through WordPress's HackerOne program. Searchlight's writeup says the attack has "no preconditions and can be exploited by an anonymous user." Cloudflare, which deployed two WAF rules in response, reported that the vulnerable code path is reachable when a persistent object cache is not in use.

NVD currently assigns a CVSS v3 base score of 7.5 (High) while the GitHub advisory classifies the severity as Critical — a gap worth noting if you triage strictly by score. As of publication Rapid7 was not aware of publicly confirmed in-the-wild exploitation, and The Hacker News reported no exploitation attempt as of July 18; Rapid7 nonetheless judges a public proof-of-concept highly likely to appear quickly, given that WordPress core is open source and both 7.0.1 and 7.0.2 sit in the public release archive.

Patch Priority

⚠ Do this first — the federal deadline is today

CVE-2026-58644 — Microsoft SharePoint Server (Subscription Edition, 2019, Enterprise Server 2016), CVSS 9.8. A deserialization-of-untrusted-data flaw allowing an unauthorized attacker to execute arbitrary code. It was weaponised as a zero-day before the July 14 Patch Tuesday fix; Microsoft later revised its bulletin to confirm in-the-wild exploitation. CISA added it to the Known Exploited Vulnerabilities catalog on July 16 with an FCEB remediation deadline of July 19, 2026 — today.

Reported post-exploitation activity includes stealing IIS machine keys and further deserialization techniques to gain persistence and deploy malware, so patching alone may not be sufficient on an instance that was already exposed — rotate machine keys and hunt before you call it closed.

Immediately behind it: update every internet-facing WordPress instance to 6.9.5 or 7.0.2 for CVE-2026-63030. WordPress is forcing updates to installs with auto-updates enabled, but has not said whether that push reaches sites that turned auto-updates off — verify the running version rather than assuming.

Threat Actor Spotlight

Data extortionSaaS credential abuse

ShinyHunters

Reporting this week describes ShinyHunters as one of the most active data-extortion operations globally, with 40+ claimed breaches in 2026 driven by social engineering and SaaS-credential abuse rather than endpoint exploitation. Researchers describe operational overlap with Scattered Spider (UNC3944) and LAPSUS$ under a "Scattered LAPSUS$ Hunters" banner run through a public Telegram channel, with a reported division of labour: Scattered Spider supplies initial access, ShinyHunters handles data theft and publication, and LAPSUS$ members amplify the extortion. The group is linked to both the Oracle PeopleSoft zero-day wave behind the Nissan disclosure and the Abbott/Exact Sciences intrusion below.

Breaches & Incidents

NewZero-dayEmployee data

Nissan discloses employee data breach tied to Oracle PeopleSoft zero-day

Nissan confirmed a breach affecting current and former employees after attackers exploited CVE-2026-35273, a critical Oracle PeopleSoft remote code execution flaw, as a zero-day between May 27 and June 9, 2026 — before Oracle shipped an emergency patch on June 10. Affected staff span the US, Canada, Mexico and Brazil, and exposed data may include contact details, banking information, Social Security numbers, Social Insurance Numbers, National Identification Numbers, financial and tax information, and dependent and beneficiary information. The notification arrived in a July 3 cluster alongside Kubota North America and Aflac's Japan subsidiary; the wider campaign has been linked to ShinyHunters, which claimed more than 100 victim organisations, mostly universities.

ExtortionHealthcare

Abbott confirms access to legacy Exact Sciences systems

Abbott confirmed unauthorized access to legacy Exact Sciences systems in its Cancer Diagnostics business after ShinyHunters listed the company. Access was obtained through vishing calls to several employees in mid-June, leading to Microsoft Entra SSO. The group's leak deadline was extended to July 21.

TakedownBotnet

Google and the FBI disrupt the NetNut residential proxy network

Google degraded NetNut — also tracked as Popa — a residential proxy service built on an estimated 2 million home devices worldwide, including smart TVs and streaming boxes. In a single week in June 2026, Google observed 316 distinct threat clusters using suspected NetNut exit nodes, spanning cybercriminal and espionage groups. The action was coordinated with the FBI, which seized some domains on July 2, and with Lumen Technologies; the network traces back to Alarum Technologies, a Nasdaq-listed Israeli company. Google assesses with high confidence that many seemingly separate proxy brands resell the same pool, so one takedown ripples across several.

Commercial spywareGovernment

European Parliament member investigating spyware was hacked with Pegasus

A member of the European Parliament who was investigating commercial spyware was themselves compromised with NSO Group's Pegasus, according to reporting carried this week.

CISA KEV & Federal Deadlines

  • CVE-2026-58644 — Microsoft SharePoint Server deserialization → RCE, CVSS 9.8, exploited as a zero-day. Added to KEV July 16; FCEB remediation due July 19, 2026.
  • CVE-2026-25089 and CVE-2026-39808 — Fortinet FortiSandbox OS command injection, exploited in the wild. Added to KEV July 16; FCEB remediation due July 19, 2026.
  • CVE-2026-46817 — Oracle E-Business Suite remote code execution, CVSS 9.8, actively exploited against roughly 950 internet-exposed instances worldwide. Covered by Binding Operational Directive 26-04 with a federal deadline of July 18, 2026.
  • Rule reminder: KEV remediation dates are now assigned per-CVE on a risk basis under BOD 26-04. The old flat "three weeks from the add date" heuristic from BOD 22-01 no longer applies — read the due date CISA publishes for each entry.
