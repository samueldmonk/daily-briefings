Top Story

wp2shell: WordPress Core falls to a pre-auth RCE chain, and the exploits arrived first

Two WordPress Core flaws chained together give an anonymous, unauthenticated attacker remote code execution against a default installation — no plugins, no login, no preconditions. CVE-2026-63030 is a REST API batch-route confusion weakness introduced in WordPress 6.9; CVE-2026-60137 is a SQL injection in the author__not_in parameter of WP_Query, present since 6.8. Chained on 6.9.0 through 7.0.1, they let an attacker reach the batch endpoint at /wp-json/batch/v1 and execute code.

WordPress shipped patches on July 17. Public proof-of-concept exploits appeared on GitHub within hours of disclosure, and Tenable reports that multiple security firms have since confirmed in-the-wild exploitation — Hexastrike saw honeypot attempts over the weekend and has assisted on confirmed incident response; Patchstack has also confirmed exploitation. WordPress.org has enabled forced automatic updates for supported installations on affected versions.

Two details make this unusual. First, the target is Core, not a plugin: Tenable notes all four prior WordPress entries in the CISA KEV catalog were plugin flaws. Second, the discovery is openly credited to AI tooling — Adam Kues of Searchlight Cyber, who found CVE-2026-63030, stated in the technical write-up that "no security researcher could have found and completed this exploit chain in 10 hours without AI." The same tooling advantage is being cited for how fast the public PoCs followed the patch.

Fixed versions: 7.0.2, 6.9.5, 6.8.6 (SQL injection only) and 7.1 beta2. Where patching must wait, Searchlight Cyber's interim options are blocking /wp-json/batch/v1 and ?rest_route=/batch/v1 at the WAF, restricting unauthenticated REST API access, or deploying its batch-endpoint plugin; Cloudflare has pushed WAF rules covering both CVEs to all plans including free. As of Tenable's July 20 update, no threat actor has been publicly attributed and no IoCs have been released.

Patch Priority

⚠ Do this first

Patch WordPress to 7.0.2 / 6.9.5 / 6.8.6 and verify the update actually landed. This is the only item on today's page that combines maximum severity, a default-configuration attack path, confirmed in-the-wild exploitation and publicly circulating exploit code. Forced auto-updates cover supported hosted installations; self-managed instances and anything with auto-update disabled will not be covered.

The hardest verified federal deadline on this page has already passed: the SonicWall SMA1000 zero-day CVE-2026-15409 (CVSS 10.0, actively exploited) carried a CISA remediation date of July 17, 2026 — 4 days overdue — with further KEV deadlines lapsing July 18 and 19. CVE-2026-63030 has been added to KEV, but no specific FCEB due date for it could be confirmed from a CISA source this run, so none is printed below.

Threat Actor Spotlight

NewRansomwareAI-operated

JADEPUFFER — the first documented end-to-end LLM-driven ransomware operation

Researchers have captured what they describe as the first complete extortion operation driven end to end by a large language model. The operator, dubbed JADEPUFFER, obtained initial access to an internet-facing Langflow instance through CVE-2025-3248 and then ran an adaptive, fully automated campaign — reconnaissance, escalation, data theft and extortion — without a human driving each step. The significance is less the payload than the operating model: agentic tooling collapses the skill floor for running a full intrusion, and the same automation advantage that produced same-day wp2shell exploits is now visible on the intrusion side. Researchers tie the operator to ENCFORGE, a newly compiled Go ransomware built to encrypt model weights, vector indexes and training datasets. Sources: Dark Reading, Sysdig.

Breaches & Incidents

NewAI infrastructure

Hugging Face breached by an autonomous AI agent

On July 20 Hugging Face confirmed that an unauthorized third party used an autonomous AI agent to breach its production infrastructure, running more than 17,000 logged actions before the intrusion was contained. Entry was a malicious dataset in the processing pipeline that chained two flaws — a remote-code dataset loader and a template-injection bug. Internal datasets and infrastructure credentials were accessed; the company is still investigating whether customer or partner data was taken. Hugging Face closed the flaws, evicted the attacker, rebuilt affected nodes, rotated the exposed credentials and urged users to rotate any keys stored on the platform. Sources: Hugging Face security disclosure, BleepingComputer, TechCrunch, The Hacker News.

Supply chain

SleeperGem backdoors three RubyGems packages

Between July 18 and 19, malicious versions of three gems were published to RubyGems.org: git_credential_manager (impersonating Microsoft's Git Credential Manager), Dendreo, and fastlane-plugin-run_tests_firebase_testlab. Each release is a loader that pulls a second stage from an attacker-controlled Forgejo host, checks for CI/CD environment variables from platforms like GitHub Actions and GitLab, and skips execution inside build systems — running only on developer machines, where it drops a native daemon and persists via cron and systemd. Two of the accounts had been dormant for years. Aikido Security's Charlie Eriksen: "A RubyGems account that has gone quiet for six or seven years doesn't look risky to anyone." Sources: The Hacker News, StepSecurity, SC Media.

Supply chainnpm

@asyncapi npm compromise — the "--ignore-scripts won't save you" pattern

Per Microsoft's Security Blog (July 15), five package versions across four @asyncapi package names were republished inside roughly 90 minutes on July 14. The payload executes at module-load time and the packages declare no lifecycle hooks at all, so npm install --ignore-scripts does not neutralise it. Root cause was a GitHub Actions "pwn request" — a pull_request_target workflow checking out an untrusted PR head while privileged — which exposed a bot PAT; the attacker then rode legitimate release workflows, so all five malicious versions carry valid provenance attestations. Pin specs 6.11.1, generator 3.3.0, components 0.7.0, helpers 1.1.0. SleeperGem shares the same property from a different angle: build-system checks are not a safety net.

RansomwareHealthcare

Conduent breach total reaches 62.2 million

The SafePay ransomware incident at Conduent now stands at 62.2 million individuals, making it the third-largest US healthcare breach on record behind Change Healthcare (192.7M, 2024) and Anthem (78.8M, 2015). Discovered January 13, 2025, with attacker access dating to October 21, 2024. The count escalated from 10.5M to 25.5M in February and to 62.2M in June 2026. Conduent booked $25M in direct response costs in Q1 2025 with a further $16M anticipated through Q1 2026. Sources: HIPAA Journal, BankInfoSecurity, Paubox.

Extortion

Extortion claims against Synopsys, Bosch and Deutsche Bank

The D1R cybercrime group claimed on July 14 to have stolen data from Synopsys and Bosch, threatening to leak it unless paid. Separately, concern over a Deutsche Bank breach rose after a ransomware group posted purported evidence on July 8. Both remain claims by the attackers — neither is treated here as a confirmed breach, and no victim confirmation was found this run. Context for the pace: Infosecurity Magazine reports a new ransomware group emerging roughly every week, 61 new groups in 2026 and 146 active groups having named at least one victim as of June. Sources: Cybernews, Infosecurity Magazine.
