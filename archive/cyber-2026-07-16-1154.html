Top Story

Microsoft patches a record 622 flaws — and two are already being exploited

Microsoft shipped its largest Patch Tuesday on record this week, fixing 622 of its own CVEs by the Security Update Guide count — more than triple June's previous high of around 200. Two of the fixes close holes attackers are already using, and Microsoft credits incident responders for finding both. They are the ones to grab first: CVE-2026-56164, an elevation-of-privilege flaw in on-premises SharePoint Server, and CVE-2026-56155, an elevation-of-privilege flaw in Active Directory Federation Services.

Neither is a splashy remote-code-execution critical, but both sit in systems that matter more than their scores suggest — the company document store and the box that signs its logins. Microsoft says the SharePoint bug lets an unauthenticated attacker escalate privileges over the network. Within hours of the release, a researcher published a proof-of-concept ("LegacyHive") for a separate Windows User Profile Service privilege bug, underscoring how fast the window between patch and exploit is closing (The Hacker News, Jul 14–15).

Patch Priority

Patch first: CVE-2026-56164 (SharePoint Server) and CVE-2026-56155 (AD FS) — both exploited zero-days

Of the 622 Microsoft fixes, prioritize the two that are already under attack: the on-premises SharePoint Server privilege-escalation flaw (unauthenticated, over the network) and the AD FS privilege flaw in the system that signs your logins. Right behind them, patch SonicWall SMA 1000 against the actively exploited CVE-2026-15409 (CVSS 10.0, unauthenticated SSRF) and CVE-2026-15410 — fixes are in the 12.4.x releases. Treat internet-facing SharePoint, AD FS and SMA 1000 appliances as priority targets today (The Hacker News, Jul 14–15).

Threat Actor Spotlight

OkoBot — malware that steals your hardware-wallet recovery phrase from inside the real app

Kaspersky's GReAT team detailed OkoBot, a Windows malware framework running since April 2025 whose SeedHunter module cons hardware-wallet owners out of their recovery phrase. On an infected PC the prompt appears inside the wallet's own desktop software — Trezor Suite, Ledger Wallet or Ledger Live — sometimes waiting until the device is plugged in; the page is malicious but the app around it is the genuine one the victim installed. OkoBot carries more than 20 payloads and implants and was still active as of the July 15 report, with hundreds of victims across 25+ countries, concentrated in Brazil, Vietnam, Canada, Mexico and Türkiye (The Hacker News, Jul 15).

Breaches & Incidents

zero-dayNew

SonicWall SMA 1000 zero-days exploited in the wild

SonicWall warned of active exploitation of two zero-days in its Secure Mobile Access 1000 appliances: CVE-2026-15409 (CVSS 10.0), an unauthenticated server-side request forgery flaw, and CVE-2026-15410 (7.2), a post-authentication code-injection bug in the Appliance Management Console that can run OS commands as administrator. Fixes ship in the 12.4.x builds; SonicWall says it has investigated multiple exploitation cases (The Hacker News, Jul 15).

KEVNew

CISA adds exploited Oracle E-Business Suite flaw

CISA added CVE-2026-46817, an improper-privilege-management flaw in Oracle E-Business Suite, to its Known Exploited Vulnerabilities catalog on July 15 after confirming active exploitation that can lead to takeover of Oracle Payments. It is one of six KEV additions in the past week — 20 in the last 30 days (CISA/gbhackers/Senserva).

supply chainNew

Compromised @asyncapi npm packages drop a botnet loader

Four packages in the @asyncapi namespace were poisoned to deliver a multi-stage botnet loader. An obfuscated first stage pulls an encrypted second-stage payload ("Miasma") from IPFS; unlike earlier attacks that used install hooks, the malicious code runs when Node.js loads the module (OX Security/SafeDep/Socket/StepSecurity via The Hacker News, Jul 15).

vulnerability

Cursor flaw runs code just from opening a repo

Open a project in the Cursor AI editor on Windows and, if a file named git.exe sits in the project root, Cursor runs it — no click, no prompt — as the logged-in user, with access to source, SSH keys and cloud tokens. Mindgard reported it in December 2025 and published details this week; there is still no patch (The Hacker News, Jul 15).

malware

LabubaRAT poses as NVIDIA software to seize Windows hosts

Blackpoint Cyber flagged a previously undocumented Rust remote-access trojan, LabubaRAT, delivered via an executable named "nvidia-sysruntime.exe." It profiles the host, identifies security tools, moves files, captures screenshots and proxies traffic, communicating over HTTPS, WebView2 and DNS tunneling — with signs it is sold as malware-as-a-service (The Hacker News, Jul 14).

browser security

Firefox and Chrome ship emergency critical fixes

Mozilla patched two critical Firefox flaws (CVE-2026-15718, CVE-2026-15719) for which public exploit code exists, fixed in 152.0.6, while Google shipped 15 Chrome fixes including two critical use-after-free bugs in the Ozone display layer. Update both browsers now (The Hacker News, Jul 15).

AI tradecraft

TuxBot v3: an IoT botnet built with an LLM's help

Palo Alto Networks Unit 42 detailed TuxBot v3 Evolution, an IoT botnet framework showing signs of LLM-assisted development — the AI even left a safety disclaimer in the generated code that the developer forgot to remove, and several functions simply didn't work (The Hacker News, Jul 15).

vulnerability

RabbitMQ flaws could leak OAuth secrets, cross tenants

Miggo disclosed two access-control flaws in the RabbitMQ message broker: one leaks the broker's confidential OAuth secret to an unauthenticated attacker in a single request (a path to full takeover), the other lets any logged-in user read other tenants' data. Present since early 2024, fixed in 4.3.0 and back-ported releases; no known exploitation before disclosure (The Hacker News, Jul 14).
